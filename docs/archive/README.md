# Documentation archive

Active documentation is intentionally small: `AGENTS.md`, `README.md`,
`PLAN.md`, `TASKS.md`, `CHANGELOG.md`, `docs/plan-research-platform.md`,
`docs/project-guardrails.md`, `docs/model-usage.md`, `docs/design.md`,
`docs/learning.md`, active validation/strategy plans and operational skills.

This directory holds historical detail that should not compete with those
canonical files:

- `plans/` — completed stage plans;
- `design/` — superseded visual references;
- `learning/` — detailed phase notes consolidated into `docs/learning.md`;
- dated changelog and review records.

Do not archive guardrails, active skills, source-grounded strategy rules,
validation evidence or the canonical research-platform plan while they remain
operationally binding. When a stage closes, preserve its stable ID, acceptance
result and archive pointer in `TASKS.md`/`CHANGELOG.md`, then search for stale
references before finishing the cleanup.
